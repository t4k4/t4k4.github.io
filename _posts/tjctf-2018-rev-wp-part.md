#### Python Reversing(40p)
> Found this flag checking file and it is quite vulnerable

题目直接给了源码，直接对算法逆向即可
源码：
```python
import numpy as np

flag = 'redacted'

np.random.seed(12345)
arr = np.array([ord(c) for c in flag])
other = np.random.randint(1,5,(len(flag)))
arr = np.multiply(arr,other)

b = [x for x in arr]
lmao = [ord(x) for x in ''.join(['ligma_sugma_sugondese_'*5])]
c = [b[i]^lmao[i] for i,j in enumerate(b)]
# str的zfill方法只补全不截断
print(''.join(bin(x)[2:].zfill(8) for x in c))

# original_output was 1001100001011110110100001100001010000011110101001100100011101111110100011111010101010000000110000011101101110000101111101010111011100101000011011010110010100001100010001010101001100001110110100110011101
```
这里有个地方要注意，给了长度为202的原始输出，每8位或9位对应原始输入的一个字符，可以根据生成的随机整数(1 or 2 or 3 or 4)判断，如果是3或4，则对应9位，否则对应8位，然后根据输出做调整。
实际写脚本的时候，只按随机数判断得到的结果出现乱码，但是大部分是正确的，只要调整后面几个字符就可以了。
脚本：
```python

output = '1001100001011110110100001100001010000011110101001100100011101111110100011111010101010000000110000011101101110000101111101010111011100101000011011010110010100001100010001010101001100001110110100110011101'
lengths = [9,8,8,8,8,8,9,9,8,9,8,8,9,8,9,9,8,9,8,9,8,8,8,9]

    arr.append(int(output[i:i+l], base=2))
    i += l

for c in arr:
    flag += chr(c)
```

#### Bricked Binary(80p)
> Earlier, I input my flag to this image and received 22c15d5f23238a8fff8d299f8e5a1c62 as the output. Unfortunately, later on I broke the program and also managed to lose my flag. Can you find it for me?

直接丢IDA，发现逻辑同样非常简单……
关键函数是`hash`
```c
int __cdecl hash(char *s)
{
  int result; // eax@3
  int i; // [sp+4h] [bp-14h]@1
  signed __int32 v3; // [sp+8h] [bp-10h]@1

  v3 = strlen(s);
  for ( i = 0; ; ++i )
  {
    result = i;
    if ( i >= v3 )
      break;
    // 用s[v3 - 1 - i]对数组v做索引，然后与u[i]异或
    // 就得到hash(?)后的s[v3 - 1 - i]
    // 这里下标需要注意一下
    s[v3 - 1 - i] = u[4 * i] ^ v[4 * s[v3 - 1 - i]];
  }
  return result;
}
```
